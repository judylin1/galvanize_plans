Authorization

Some notable quotes:

From Derek B:

Can be compared to airport; authentication is the TSA agent checking your id to make sure you are who you say you are, whereas authorization is the person at the gate presumes you've already been authenticated by the TSA agent and just has to authorize whether or not your ticket garners you access to that specific flight.

From Jaylyn:

Authorization is handled with middleware with router.use() and router.get() or .post(), along with the "next" and "err" parameters that render different pages if certain criteria are met (or not met).

You can install third-party middleware in addition to your own with npm install and app.use().

Authorization != authentication: "authentication is knowing who an entity is, while authorization is knowing what a given entity can do."

There are a LOT of ways to mess it up: http://www.toptal.com/security/10-most-common-web-security-vulnerabilities

Middleware Objectives

By the end of this lesson you should be able to:

  • Use application-level middleware to set locals that will be available to all view templates
  • Use application-level middleware to redirect all unauthenticated requests to all routes that appear "after" that middleware
  • Use router-level middleware to redirect all unauthorized requests to all routes that use that middleware
  • Describe what Express middleware is and how next() works
  • Draw a middleware chain when given an express app (including both application-level and route-level middleware)
  • Describe what the body-parser and cookie-parser middleware do

Guiding questions:

  • What does next do?
  • What happens if you don't call it?
  • Does order matter when adding middleware? Why or why not?
  • What is the difference between a route and middleware?
