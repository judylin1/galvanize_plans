Tic-tac-toe

Objectives

This is a combination Cognitive and Non-Cognitive exercise.

On the cognitive side your goal is to:

  • Reason your way through each problem, engaging your "slow thinking" brain

On the non-cognitive side your goals are:

  • To "wrestle with it", and have some fun! Don't be afraid to take risks and break things. It's only code :)
  • To be an effective pair
    • Favor equal driving time to being a keyboard hog
    • Favor suggestions ("what if...") over declarations ("we should...")
    • Favor leading your pair to a solution (non-leading questions) over assuming the role of a teacher / lecturer
    • Favor having an awesome process of pairing / debugging / being methodical, over just having a working product (no need to rush)
  • To be disciplined about keeping track of the challenges you are encountering

Exercise

Your mission, should you choose to accept it, is to implement a fully functioning tic-tac-toe board in javascript and CSS that can be playable by two people at the same computer.

Requirements

Users should see a 3x3 grid on the screen.

When a user clicks an empty cell, it should place an X in the cell. When a user clicks another empty cell, it should place a Y in the cell and it should alternate back and forth until the game is over.

When a user clicks in a cell that's already populated, nothing should happen.

When a winning move is played, show an alert that displays the winning player in the modal. After clicking OK, a new game should start.

When a game move is made that casts the game to a draw, show an alert that displays "Draw", and reset the game after the user clicks "OK".

Use CSS and client-side javascript to make this happen.

Break it down

  • What questions do you have?
  • Break the problem down - what smaller problems will you need to solve?
  • What don't you know? What technical issues do you think you'll have to review / lookup?

Build it

Work in pairs.

  • Straight HTML + JavaScript files are OK, or you could build an Express app and practice that more
  • jQuery / pure JavaScript / any JavaScript framework - all OK
  • Hand-rolled CSS / Foundation / Bootstrap - all OK

Reflect

  • What problems did you encounter that you didn't anticipate?
  • On the learn-to-learn skills (setting the stage, playing), how did you do?
  • What new questions do you have now that you've done this exercise?

Stretch it out!

Instead of having two people play at the same computer, make it so that the computer can play you!

To start, have your algorithm just pick a random spot. Then evolve your algorithm until your code uses optimal game play. For example, your code would:

  • Favor a winning move if there is one
  • Favor blocking your opponent if possible
