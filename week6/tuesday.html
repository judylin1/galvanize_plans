Morning - Validating user input

Forms

What good is a form if it allows users to submit empty or bad data?

Let's practice adding server side validations to our forms to ensure our data will always be good.

Objectives

You should be able to:

  • Validate form data before inserting it into you database
  • Display communicative error messages when form data is invalid
  • Display success messages when data has been successfully added
  • DRY (do not repeat yourself) up your code by using one template to display duplicate information in your app (i.e. forms, error messages)
