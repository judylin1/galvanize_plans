Cookies!!

By the end of this lesson you should be able to:

  • Explain cookies (how they are sent from servers to browsers, and from browsers to servers)
  • View and delete cookies in Chrome developer tools
  • Set and read cookies in Express
  • Explain what incognito mode / profiles are and use them to test the same app with different cookies

Research

Research these questions (and we'll call on you randomly for answers):

  • What tab do you go to in order to see / delete cookies?
  • Are all cookies sent to all websites?
  • Do all cookies stay around when you close your browser?
  • Do cookies stay on your machine forever?
  • How are cookies transmitted to the server?
  • How are cookies transmitted to the browser?
  • How much data can you store in a cookie?

Setting cookies

Example Hit Counter

router.get('/', function(req, res, next) {
  var visits = parseInt(req.cookies.visits, 10) || 1
  visits++
  res.cookie('visits', visits);
  res.render('index', { title: 'Express', visits: visits });
});

Exercise

Create a super basic fake login with cookies.

  1. Create a new express app
  2. On the index route, create a simple form that posts to a route with the path /fake-login
  3. Add a text field in the form with the name user_name and add a submit button
  4. When the user posts to /fake-login, set a cookie named currentUser and set the value to whatever the user entered
  5. Add code to make the message "Hello [User]" appear
  6. Add a logout link that deletes that cookie
  7. Go to an incognito tab or separate profile and test again

Patterns

  • Express parses the route and creates an object representing the path params
  • Express parses the body and creates an object representing the form params
  • Express parses the querystring and creates an object representing the query params

What does Express do with the cookie header? It creates an object.
