Introduction to JavaScript

JavaScript was introduced in 1995 as a way to add programs to web pages in the Netscape Navigator browser. It was created in just 10 days by Brendan Eich

It is important to note that JavaScript has almost nothing to do with the programming language named Java. The similar name was a marketing move intended to ride the coattails of the Java programming language, which at the time was being heavily marketed and was gaining popularity.

Why Should YOU Care About Javascript

The flexbility of JS leaves space for a lot of techniques that are impossible in more rigid languages.

Javascript allows us to make our pages interactive, dynamic and awesome. Here are some very common uses for JS on the front-end:

  • Simple Navigation Menu
  • Overlay Effects
  • Animated Border Menus

Some other more complex uses for JS:

  • Interactive Music Video
  • Patatap
  • Jam With Chrome
  • Arcade Fire Reflektor Music Video

Values and Expressions

Values are the simplest components in JavaScript. 1 is a value, true is a value, "hello" is a value, function() {} is a value.

Types of values like Number or String are not very useful without being able to form Expressions or Combinations.

Try your favorite number operators

  1 + 1
  => 2
  2 - 1
  => 1

You can also create expressions with strings using addition

  "Hello, " + "world!"
  => "Hello, world!"

This is called String Concatentation.

undefined & null

undefined: Represents a value that hasn't been defined

A variable that has not been assigned a value is of type undefined. A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value. A function returns undefined if a value was not returned.

null: Represents an explicitly empty value

The DOM

The Document Object Model (DOM) is the interface between your HTML, CSS, and Javascript

It provides a structural representation of the document, enabling the developer to modify its content and visual presentation. Essentially, it connects web pages to scripts or programming languages.

The DOM specifies how browsers should create a model of an HTML page and how JS can access and update the contents of a web page while it is in the browser window.

  • The model (the DOM tree) is made of objects. Each object represents a different part of the page loaded in the browser window.

  • These objects are often referred to as nodes. Each node is an object with methods and properties. (EXAMPLE: getElementById, innerHTML)

Accessing and updating the DOM tree involves 2 steps:

  1. Locate the node that represents the element you want to work with.
  2. Use its text content, child elements, and attributes to achieve the desired results

HINT: Use a variable to store a reference to the object in the DOM tree (the location of a node).

Functions

A function is a JavaScript procedureâ€”a set of statements that performs a task or calculates a value.

A JavaScript function is written like this:

function myFunc() {
  return "hello";
};

You can also write an "anonymous" function (meaning, it has no name) like this:

function() {
  return "Iago is learning JavaScript!";
}

3 important things to remember about functions:

  1. If you want the function to return a value, you have to use the return keyword.

  2. A function doesn't run (or execute) until it is "called". To call a function you write the name of the function followed by opening and closing parenthesis and a semicolon

  3. To call the above function would look like this:

myFunc();

It might be helpful to think of a function like a recipe. A recipe consists of a list of ingredients as well as step by step instructions that tell a person what to do with the ingredients. Like, to bake a cake, for example, the recipe might look like this:

  • Mix one cup of flour with 6 eggs
  • Add one cup of milk
  • Stir until there are no clumps
  • Pre-heat oven to 425 degrees
  • Poor batter into pan

Well, a function is similar in that it contains information about how to execute a specific task. Functions are broken down into step by step instructions that a computer will execute. Functions use things like variables to store values that can be used, along with expressions and statements, to achieve a result.

But there is one important thing to remember here. Unlike humans, computers don't actually know anything unless we tell them.

Take the above example. The first line in our recipe says "Mix one cup of flour with six eggs". A human is able to figure out where the milk and eggs are on their own. Heck, a human even knows what milk and eggs are without being told in the recipe.

If we were to tell a computer to bake a cake, we would have to add a lot more steps to our recipe. For example, we would have to first define milk and eggs. We would even have to tell the computer to walk to the refrigerator, open the door, pull out the milk etc ...
